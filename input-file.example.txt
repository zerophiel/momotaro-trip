# ============================================================================
# ITEM FORMATS - Price Formats Supported
# ============================================================================

# Format 1: Ribu (rb) - thousands
Product A 125rb
Product B 195rb
Product C 285rb

# Format 2: Juta (jt) - millions
# Note: 3,4jt = 3.400.000 (comma as decimal separator)
Expensive Product 3,4jt
Another Product 1.5jt
Premium Item 2,8jt

# Format 3: Standard number format with dots
Standard Product 1.989.000
Another Standard 2.289.000
High Value Item 5.399.000

# Format 4: Price in the middle of text (first price found will be used)
# Example: "Product name 285rb additional info" -> price = 285rb
Product with price in middle 285rb some additional text here
Another example 125rb with more text after price

# Format 5: Mixed format (price can appear anywhere)
Complex Product Name 439rb isi 40 pcs additional info
Item Description 1.989.000 with more details

# ============================================================================
# CUSTOMER FORMATS - Checkbox and Numbered List
# ============================================================================

# Format 1: Checked checkbox [x] - WILL BE BILLED
Product A 125rb
- [x] Customer A +62 812-XXXX-XXXX
- [x] Customer B +62 813-XXXX-XXXX

# Format 2: Unchecked checkbox [ ] or [] - WILL NOT BE BILLED
Product B 195rb
- [ ] Customer C +62 814-XXXX-XXXX
- [] Customer D +62 815-XXXX-XXXX

# Format 3: Numbered list (1., 2., 3., etc.) - WILL NOT BE BILLED
Product C 285rb
1. Customer E +62 816-XXXX-XXXX
2. Customer F +62 817-XXXX-XXXX
3. Customer G +62 818-XXXX-XXXX

# Format 4: Dash without checkbox - WILL NOT BE BILLED
Product D 195rb
- Customer H +62 819-XXXX-XXXX

# ============================================================================
# NOTES FOR CONTRIBUTORS
# ============================================================================
#
# 1. Price Parsing:
#    - Priority: rb/jt patterns first, then long number format
#    - First valid price found in line will be used
#    - Price will be removed from item name automatically
#
# 2. Customer Identification:
#    - Primary: Phone number (normalized: +62 and 0 are equivalent)
#    - Secondary: Name (case-insensitive) if no phone number
#    - Same customer = same phone OR (no phone AND same name)
#
# 3. Quantity Calculation:
#    - Each repeated entry for same item = +1 quantity
#    - Only entries with [x] are counted
#
# 4. Phone Number Normalization:
#    - +62 XXX-XXXX-XXXX and 08XXXXXXXXXX are treated as same
#    - All phone numbers are normalized internally for comparison
#    - Display format: +62 XXX-XXXX-XXXX
#
# 5. Text Cleaning:
#    - "ok" at the end is removed (case-insensitive)
#    - Text in parentheses is removed
#    - Unicode characters are normalized
#
# 6. Special Sections:
#    - "Product REQUEST cek harga" section is completely ignored
#    - All items and customers below this line are skipped
#
# ============================================================================
